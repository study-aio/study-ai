## 1. 데이터 분석과 기초 통계 (a_intro)

### Data Analysis
- 원시(원본 그대로) 데이터를 분석하여 인사이트(가시성 증가 및 깊은 이해)로 변환하는 작업이다.
- 문제를 해결하기 위해 데이터를 사용해서 흐름 및 방향을 찾는 기술이다.
- 데이터 분석을 통해 비지니스 프로세스를 구성하고, 의사 결정을 개선하며, 비지니스 성장을 증진할 수 있다.
- ETL은 Extract(추출), Transform(변환), Load(적재)를 의미하며, 여기 저기 흩어진 데이터를 하나로 모으기 위한 결합 과정이다.

### 기초 통계 (Basic statistics)
- 통계는 아직 발생하지 않은 일을 예측하기 위해 사용한다.
- 수량적인 비교를 기초로 많은 사실을 관찰하고 처리하는 방법을 연구하는 학문이다.
- 불균형 데이터를 대상으로 규칙성과 불규칙성을 발견한 뒤 실생활에 적용할 수 있다.

### 기초 통계 용어 정리

| 용어 | 설명 |
| --- | --- |
| 변량(Variable) | 자료의 수치, 즉 데이터의 값 |
| 계급(Class) | 변량을 일정 간격으로 나눈 구간 (소괄호 미포함, 대괄호 포함) |
| 도수(Frequency) | 각 계급에 속하는 변량의 개수 |
| 상대 도수 | 각 계급에 속하는 변량의 비율 |
| 도수분포표 | 자료를 계급별로 나누어 도수 및 상대 도수를 조사한 표 |
| 히스토그램 | 도수분포표를 시각화한 그래프 |

### 대표값과 산포도

| 용어 | 설명 |
| --- | --- |
| 산술 평균(Mean) | 변량의 합을 변량의 수로 나눈 값 |
| 편차(Deviation) | 변량에서 평균을 뺀 값 (편차의 합은 항상 0) |
| 분산(Variance) | 편차를 제곱하여 합한 뒤 산술 평균을 낸 값 |
| 표준편차(Standard deviation) | 분산의 제곱근. 흩어진 정도를 하나의 수치로 나타냄 |

### 확률변수 (Random variable)
- 머신러닝, 딥러닝 등 확률을 다루는 분야에서 필수적인 개념이다.
- 시행(trial)을 해서 어떤 사건(event)이 나타났는지에 따라 값이 정해지는 변수이다.
- 알파벳 대문자(X, Y, Z)로 표현하며, 원소(확률변수값)는 소문자(x, y, z)로 표현한다.

### 확률변수 종류

| 구분 | 설명 | 예시 |
| --- | --- | --- |
| 범주형 확률변수 | 수치가 아닌 기호/언어로 표현. 유한집합 | {앞면, 뒷면}, {봄, 여름, 가을, 겨울} |
| 이산형 확률변수 | 수치로 표현하고 셀 수 있는 값 | {0, 1, 2, 3}, {10, 20, 30} |
| 연속형 확률변수 | 구간을 나타내는 수치. 셀 수 없는 무한집합 | 128.56 < X < 268.56 |

### 확률분포 (Probability distribution)
- 사건에 대한 확률변수에서 정의된 모든 확률값의 분포이며, 서로 다른 모든 결과의 출현 확률을 제공한다.
- 동전 던지기, 주사위 던지기처럼 시행에서 얻는 결과들의 확률을 표현한다.

### 확률분포함수 (Probability distribution function)
- 확률변수의 분포를 나타내는 함수로서, 확률변수값이 나올 확률을 나타내는 함수이다.
- 확률질량함수, 확률밀도함수 등이 있다.

#### 확률질량 함수 (Probability mass function, pmf)
- 확률변수 X의 분포를 나타내는 함수로서, x가 나올 확률이다.
- 범주형 확률변수와 이산형 확률변수에서 사용된다.
- 확률변수에서 각 값에 대한 확률을 나타내는 것이 마치 각 값이 "질량"을 가지고 있는 것처럼 보이기 때문에 확률질량 함수로 불린다.

#### 확률밀도 함수 (Probability density function, pdf)
- 확률변수 X의 분포를 나타내는 함수로서, 특정 구간에 속할 확률이며 이는 특정 구간을 적분한 값이다.
- 연속형 확률변수에서 사용된다.
- 구간에 포함될 확률(밀도)을 나타내기 때문에 확률밀도 함수라 불린다.

### 미적분 기초

| 용어 | 설명 |
| --- | --- |
| 무한대(Infinity) | 끝없이 커지는 상태. 기호 ∞ |
| 무한소 0(Infinitesimal) | 0에 매우 근접하지만 0이 아닌 상태 |
| 미분(Differential) | 독립변수가 미세하게 변화할 때 종속변수의 순간 변화량 |
| 적분(Integral) | 무한소 0인 사각형(선분)을 쌓아 면적을 구하는 것 |

### 정규분포 (Normal distribution)
- 모든 독립적인 확률변수들의 평균은 어떠한 분포에 가까워지는데, 이 분포를 정규분포라고 한다.
- 비정규분포의 대부분은 극한상태에 있어서 정규분포에 가까워진다.
- 평균 μ(mu)와 표준편차 σ(sigma)에 대한 확률밀도함수를 가지는 분포를 의미한다.

### 표준 정규분포 (Standard normal distribution)
- 정규분포는 평균과 표준편차에 따라 모양이 달라지므로 일반화할 수 없다.
- N(μ, σ) = N(0, 1)로 변환하면 모두 같은 특성을 가지는 동일한 확률분포로 일반화할 수 있다.

### 표준화 (Standardization)
- 다양한 형태의 정규분포를 표준 정규분포로 변환하는 방법이다.
- 표준 정규분포에 대한 값(넓이)을 이용해 원래 분포의 확률을 구할 수 있다.

### 모집단과 모수 (Population and population parameter)
- 모집단이란, 정보를 얻고자 하는 대상의 전체 집합을 의미한다.
- 모수란, 모집단의 수치적 요약값(평균, 표준편차 등)을 의미한다.

## 2. NumPy (b_numpy)

### Numpy
- 머신러닝 애플리케이션에서 데이터 추출, 가공, 변환과 같은 데이터 처리 부분을 담당한다.
- 넘파이 기반의 사이킷런을 이해하기 위해서는 넘파이는 필수이다.
- 기본 문법과 중요 API만 이해하는 것이 전략적으로 좋다.

### NumPy 핵심 API 요약

| 항목 | 설명 |
| --- | --- |
| ndarray | N차원 배열 객체. 반드시 같은 자료형의 데이터만 담아야 함 |
| astype() | 요소의 타입 변환. 대용량 처리 시 메모리 절약 목적 |
| axis | 축의 방향성 표현 |
| arange / zeros / ones | 연속값, 0, 1로 초기화된 배열 생성 |
| reshape() | ndarray의 기존 shape를 다른 shape로 변경 |
| Sorting | 기본 오름차순. 내림차순은 `[::-1]` 사용 |

### Indexing
- 특정 위치의 데이터를 가져오는 것
- 위치 인덱싱(Location indexing)
- 슬라이싱(Slicing)
- 팬시 인덱싱(Fancy indexing)
- 불린 인덱싱(Boolean indexing)

### 벡터와 선형대수

#### 벡터
- 데이터 과학에서 벡터란 숫자 자료를 나열한 것을 의미한다.
- 벡터는 공간에서 한 점을 나타낸다.
- feature 1개당 1차원이고, feature가 3개면 3차원이다.

#### 내적 (Dot product)
- 두 벡터의 성분들의 곱의 합이다.
- 앞 행렬의 열 개수와 뒤 행렬의 행 개수가 일치해야 한다.

#### 선형대수 (Linear Algebra)
- 선형 방정식을 풀기 위해 배우는 학문이다.
- 미지수가 n개이면 방정식도 n개가 필요하다.
- 연립 방정식을 쉽게 표현하기 위해 선형대수를 사용한다.

#### 과결정계 (Overdetermined system)
- 미지수보다 방정식이 더 많이 있는 연립방정식으로, 보통 해가 존재하지 않거나 유일하지 않다.
- 차원을 넘어간 평면에서는 교점과 가장 가까운 점을 찾아 투영을 통해 근사치를 구해야 한다.
- 투영 시 원본 값에서 어느정도 loss(손실)가 발생하지만 근사값을 구하기 위해 수행한다.

## 5. RFM 분석 (f_rfm)

### RFM 분석
- 사용자별로 얼마나 최근에, 얼마나 자주, 얼마나 많은 금액을 지출했는지에 따라 사용자 분포를 확인하거나 그룹(등급)을 나누는 분석 기법이다.
- 구매 가능성이 높은 고객을 선정할 때 용이한 데이터 분석 방법이다.
- 사용자들의 구매 패턴을 기준으로 분류를 진행하기 때문에 각 사용자 그룹의 특성에 따라 차별화된 마케팅 메시지를 전달할 수 있다.

### RFM 3대 지표

| 지표 | 설명 |
| --- | --- |
| Recency | 얼마나 최근에 구매했는가 |
| Frequency | 얼마나 자주 구매했는가 |
| Monetary | 얼마나 많은 금액을 지출했는가 |

### RFM 적용 예시 (이커머스)
- VIP 고객: 구매 횟수가 많고 금액이 크며 최근에 구매한 고객 → 연말 선물 등으로 충성도 유지
- 휴면 고객: 오래 전 한두 번 소액 구매한 고객 → 할인 쿠폰 등으로 재구매 유도
- RFM을 통해 사용자 특성별로 다른 정책을 적용하고 서비스를 더 잘 사용하도록 유도하는 전략을 세울 수 있다.

### 정규화 (Normalization)
- 값의 범위를 0 ~ 1 사이로 변환시켜서 모든 컬럼의 데이터가 평등해진다.
- 서로 다른 단위의 값(80kg과 180cm 등)은 비교 대상이 될 수 없으므로 정규화를 사용한다.
- 이를 스케일링이라고 한다.

### 마케팅 전략 제안 예시
- 웹 방문 횟수가 낮은 'Diamond', 'VIP' 등급 고객 대상 → 웹 사이트 사용성 향상 캠페인으로 이탈 방지
- 연령이 상대적으로 낮은 'Bronze', 'Silver' 등급 고객 대상 → 현대적인 제품/서비스 홍보
- 교육 수준이 높은 고객 대상 → 전문적이고 깊은 지식을 요구하는 콘텐츠 제공
- 결혼 및 자녀 여부에 따라 타겟팅된 광고 캠페인 구성

## 6. Machine Learning (g_marchine_learning)

### 6.1 ML 입문 (a_intro)

#### AI (Artificial Intelligence)

| 구분 | 설명 |
| --- | --- |
| Rule-based AI | 전문가가 직접 규칙(if-then)을 정의. 투명성 높음, 유연성 낮음 |
| Machine Learning AI | 데이터를 기반으로 규칙(패턴)을 학습하여 결과 추론 |

#### Rule-based AI 특징
- 광범위한 데이터 수집/교육이 필요 없으며 전문가 기존 지식 기반으로 비지니스 규칙 정의 가능
- 명시적인 "if-then" 규칙에 의존하므로 높은 투명성 제공
- 정적이며 유연성이 없어 사전 정의된 규칙을 수동 조정해야 함
- 모호하거나 불확실한 상황에서 어려움을 겪을 수 있음
- 전문가 개인의 편견이 들어갈 수 있어 불공평하거나 부정확한 평가로 이어질 수 있음

#### Machine Learning AI 특징
- 데이터 마이닝, 음성 인식, 영상 인식, 자연어 처리 등에서 적용된다.
- 데이터의 패턴을 학습하여 예측 등을 수행한다.

#### 머신러닝의 종류

| 종류 | 설명 | 세부 분류 |
| --- | --- | --- |
| 지도 학습(Supervised) | 문제(Feature)와 답(Target)을 함께 제공 | 분류(Classification), 회귀(Regression) |
| 비지도 학습(Unsupervised) | 문제만 제공하고 답은 제공 안 함 | 군집화(Clustering), 차원 축소 |
| 강화 학습(Reinforcement) | 행동에 따른 보상/벌점으로 학습 | 게임, 주식 투자, 자율주행 |

#### Machine Learning의 단점
- 데이터에 의존적이다 (Garbage In, Garbage Out)
- 학습 데이터로 잘 만들어진 모델도 실제 데이터에서 정확한 결과가 안 나올 수 있다.
- 결과가 왜 이렇게 나왔는지에 대한 분석이 쉽지 않다(블랙박스).
- 데이터를 넣으면 원하는 대로 좋은 결과를 얻기란 쉽지 않다.

#### R vs Python

| 언어 | 적합한 사용자 / 강점 |
| --- | --- |
| R | 개발에 익숙하지 않지만 통계 분석에 능한 현업 사용자. 다양한 통계 패키지 |
| Python | 직관적인 문법, 객체지향 함수형, 다양한 라이브러리, 상용화 용이 |

→ AI 컨텐츠에서의 머신러닝과 딥러닝은 Python을 사용한다.

### 6.2 분류 (Classifier) (b_classifier)

#### 분류 (Classifier)
- 대표적인 지도학습 방법 중 하나이며, 다양한 문제와 정답을 학습한 뒤 별도의 테스트에서 정답을 예측한다.
- 이진 분류(Binary Classification): 정답이 0(음성)과 1(양성) 두 가지
- 다중 분류(Multiclass Classification): 정답이 3개 이상

#### 핵심 용어

| 용어 | 설명 |
| --- | --- |
| 피처(Feature) | 데이터 세트의 일반 컬럼. 타겟을 제외한 나머지 속성 |
| 레이블/클래스/타겟/결정 | 지도 학습 시 정답으로 주어지는 값 |

#### 분류 예측 프로세스

| 단계 | 사용 함수/메서드 | 설명 |
| --- | --- | --- |
| 데이터 세트 분리 | train_test_split(feature, target, test_size, random_state) | 학습/테스트 데이터 분리 |
| 모델 학습 | fit(train_feature, train_target) | 모델 학습 |
| 예측 | predict(X_test) | 새로운 데이터 예측 |
| 평가 | accuracy_score(y_test, predict(X_test)) | 정확도 평가 |

### 결정 트리 (Decision Tree)
- 매우 쉽고 유연하게 적용될 수 있는 알고리즘으로, 데이터 전처리 의존도가 매우 적다.
- 학습을 통해 데이터에 있는 규칙을 자동으로 찾아내서 Tree 기반의 분류 규칙을 만든다.
- 각 특성이 개별적으로 처리되어 데이터 스케일의 영향을 받지 않으므로 정규화/표준화 같은 전처리가 필요 없다.
- 영향을 가장 많이 미치는 feature를 찾아낼 수 있다.
- 복잡한 규칙 구조를 가지면 과적합(Overfitting)이 발생할 수 있다.

#### 트리 노드 구조

| 노드 종류 | 설명 |
| --- | --- |
| 루트 노드 | 가장 상위 노드 |
| 서브 노드 | 나머지 분기점 |
| 리프 노드 | 결정된 분류값 노드 |

#### 지니 계수(Gini coefficient)
- 정보의 복잡도를 수치화한 값으로 0과 1 사이의 값을 가진다.
- 클래스가 섞이지 않고 분류가 잘 되었다면 불순도가 낮다(지니 계수 낮음).
- 지니 계수가 낮을수록 분류가 잘 된 것이다.

#### 결정 트리 하이퍼파라미터

| 파라미터 | 용도 |
| --- | --- |
| max_depth | 최대 깊이 제한으로 모델 단순화 |
| max_features | 분할에 고려할 feature 수 (None, sqrt, log2, 정수, 실수) |
| min_samples_split | 분할에 필요한 최소 샘플 수 |
| min_samples_leaf | 리프 노드의 최소 샘플 수 (과적합 방지에 특화) |
| class_weight | 데이터 불균형 시 특정 클래스에 가중치 부여 (None, balanced, dict) |

### 교차 검증(Cross Validation)
- 학습 데이터를 다시 분할하여 학습 데이터와 모델 성능을 1차 평가하는 검증 데이터로 나눈다.
- 여러번 검증해서 정확도라는 수치를 더 신뢰할 수 있게 된다.

#### 교차 검증 장단점

| 구분 | 내용 |
| --- | --- |
| 장점 | 특정 데이터 세트에 대한 과적합 방지, 검증 데이터 분리로 인한 과소적합 방지 |
| 단점 | 모델 훈련, 모델 평가에 소요되는 시간 증가 |

목적: 과적합을 피하고 하이퍼 파라미터를 튜닝함으로써 모델을 일반화하고 신뢰성을 증가시키기 위해 사용한다.

#### 교차 검증의 종류

| 종류 | 설명 |
| --- | --- |
| K-Fold | k개의 폴드 세트로 k번 학습/검증 반복. 타겟 비중이 치우칠 수 있음 |
| Stratified K-Fold | K-Fold + 타겟 분포도가 유사하도록 분할. 데이터 치우침 방지 |

#### 하이퍼파라미터 튜닝

| 도구 | 설명 |
| --- | --- |
| cross_val_score | 간편한 교차 검증 점수 계산. classifier는 내부적으로 Stratified K-Fold 사용 |
| RandomizedSearchCV | 제시된 파라미터 중 무작위로 n_iter번 실행 |
| GridSearchCV | 제시된 파라미터의 모든 경우의 수 실행 |

### 분류 성능 평가 지표

#### 정확도(Accuracy)
- 직관적으로 모델 예측 성능을 나타낼 수 있다.
- 2진 분류 시 타겟 분포가 불균형하면 적합한 평가 지표가 될 수 없다.

#### 오차 행렬(Confusion Matrix)
- 학습된 분류 모델이 예측을 수행하면서 얼마나 예측에 오류가 있는지 함께 보여주는 지표이다.
- TP, TN, FP, FN, 4가지로 구성된다.
- 오차행렬로 분류 성능 평가 지표의 모든 것을 구할 수 있다.

| 항목 | 의미 |
| --- | --- |
| TN(True Negative) | 예측 0, 실제 0 |
| FP(False Positive) | 예측 1, 실제 0 (Error I) |
| FN(False Negative) | 예측 0, 실제 1 (Error II) |
| TP(True Positive) | 예측 1, 실제 1 |

#### 의료 진단에서의 활용
- 특이도(Specificity): 정상을 정상으로 진단하는 비율. 재현율 낮아지고 정밀도 높아질 수 있음
- 민감도(Sensitivity): 질병이 있는 사람을 질병이 있다고 진단하는 비율. 재현율 높아지고 정밀도 낮아질 수 있음

#### 평가 지표 요약표

| 지표 | 의미 | 상승 조건 |
| --- | --- | --- |
| 정밀도(Precision) | 양성 예측 중 실제 양성 비율 | TP 증가, FP 감소 |
| 재현율(Recall) | 실제 양성 중 양성 예측 비율 | TP 증가, FN 감소 |
| F1 Score | 정밀도와 재현율의 조화평균 (0 ~ 1) | 두 지표 모두 균형 있게 높음 |
| ROC AUC | ROC 곡선 밑 면적 (0 ~ 1) | 1에 가까울수록 좋음 |

#### 정밀도와 재현율의 트레이드 오프(Trade-off)
- 분류 시 결정 임계값(Threshold)을 조정해서 정밀도 또는 재현율의 수치를 높일 수 있다.
- 임계값을 낮추면 Positive로 예측할 확률 증가, 재현율 상승
- 임계값을 높이면 Negative로 더 많이 예측, 정밀도 상승
- 한 쪽 수치를 강제로 높이면 다른 한 쪽은 떨어진다.

#### F1 Score
- 이진 분류 모델의 성능을 평가하는 지표로 정밀도와 재현율의 조화평균이다.
- 산술 평균은 큰 데이터에 높은 가중치, 기하 평균은 작은 데이터에 낮은 가중치, 조화 평균은 전체적으로 동일한 가중치가 부여된다.
- 텍스트 분류, 정보 검색, 암 진단 등에서 사용된다.
- 보통 F1 Score가 0.7 이상일 때 좋은 모델이라 간주한다.

#### ROC Curve, AUC
- ROC(Receiver Operating Characteristic): FPR이 변할 때 TPR이 어떻게 변하는지 나타내는 곡선
- AUC(Area Under the Curve): ROC 곡선 밑의 면적. 1에 가까울수록 좋다.
- ROC Curve가 좌상단 모서리에 가까울수록 좋은 성능이다.

### 베이즈 추론, 베이즈 정리 (Bayesian Inference)
- 역확률(inverse probability) 문제를 해결하기 위한 방법으로, 조건부 확률 P(B|A)을 알고 있을 때 정반대인 P(A|B)을 구하는 방법이다.
- 추론 대상의 사전 확률과 추가 정보를 기반으로 "사후 확률"을 추론하는 통계적 방법이다.
- 무정보 사전확률(Non-informative Prior)에서 시작해 데이터가 수집됨에 따라 확률을 업데이트해 나가는 것이 핵심 장점이다.
- 사회적 통계나 주식에서 베이즈 정리 활용이 필수로 꼽힌다.

#### 일반 모델 vs 나이브 베이즈
- 일반 모델: 데이터(X)를 보고 정답(y)을 구분짓는 경계선을 찾음 (X에 대한 y 훈련)
- 나이브 베이즈: 정답(y)별로 모아놓고 X 특징의 확률을 계산 (y에 대한 X 훈련)

### 나이브 베이즈 분류 (Naive Bayes Classifier)
- 텍스트 분류를 위해 전통적으로 사용되는 분류기로 준수한 성능을 보인다.
- 베이즈 정리에 기반한 통계적 분류 기법으로 정확성도 높고 대용량 데이터에 대한 속도도 빠르다.
- 반드시 모든 feature가 서로 독립적이어야 한다.
- 감정 분석, 스팸 메일 필터링, 텍스트 분류, 추천 시스템 등에서 활용된다.

#### 나이브 베이즈 모델 종류

| 모델 | 특징 / 적합한 상황 |
| --- | --- |
| MultinomialNB | 가장 대중적. 데이터의 출현 빈도(Counts)를 다룰 때 최적 |
| BernoulliNB | 단어의 등장 유무(0/1)만 따짐. 짧은 텍스트(SMS, 한 줄 리뷰)에 유리 |
| GaussianNB | 연속적인 수치(연속형 데이터). 데이터 양이 적어도 효과적, 학습 속도 빠름 |

#### 기타 보조 개념

| 개념 | 설명 |
| --- | --- |
| 언더 샘플링 | 타겟 비중이 가장 낮은 개수로 다른 타겟 비중을 맞추는 작업 |
| CountVectorizer | 문장의 단어에 인덱스를 붙여 각 단어의 빈도수를 세는 기술 |
| alpha (Laplace smoothing) | 학습에 없던 단어로 확률이 0이 되는 것을 방지 |

### 6.3 회귀 (Regression) (c_regression)

### 회귀 (Regression)
- 데이터가 평균과 같은 일정한 값으로 돌아가려는 경향을 이용한 통계학 기법이다.
- 여러 개의 독립 변수와 한 개의 종속 변수 간의 상관관계를 모델링하는 기법을 통칭한다.
- feature와 target 데이터 기반으로 학습하여 최적의 회귀 계수(W)를 찾는 것이 목적이다.

#### 가설(Hypothesis)과 손실 함수
- 학습 데이터를 가장 잘 표현할 수 있는 직선을 찾아야 하고, 이 식을 가설(Hypothesis)이라 한다.
- 수학에서 W는 기울기, b는 절편이지만, 머신러닝에서는 W(Weight) 가중치, b(bias) 편향이라 한다.
- 오차의 부호를 없애기 위해 제곱하여 평균을 구하며 이를 평균 제곱 오차(MSE)라 한다.
- 손실 함수가 최소가 되는 W와 b를 구함으로써 훈련 데이터를 가장 잘 나타내는 직선을 선택한다.

#### 경사 하강법 (Gradient Descent)
- Loss 값이 최소가 되는 W를 찾기 위해 임의의 초기값에서 점차 W를 수정하는 알고리즘이다.
- Loss 값이 최소가 되는 지점은 미분값이 0이 되는 지점이다.
- 학습률(η, 에타): 하강 속도 조절. 너무 크면 발산, 너무 작으면 최소값까지 도달하지 못함

#### R² (결정 계수)
- 전체 변동성 중 모델이 해결하지 못한 오차의 비율을 뺀 값이다.
- 내 모델이 평균이라는 기준점보다 얼마나 더 똑똑해졌는지를 나타내는 척도이다.
- 분모는 평균만 알 때 발생하는 오차, 분자는 학습 후 발생한 오차이다.

### 다변량 선형 회귀 (Multivariate Linear Regression)
- 하나의 종속변수와 여러 독립변수 사이의 관계를 분석하는 기법이다.

#### 다중 공선성과 VIF

| 개념 | 설명 |
| --- | --- |
| 다중 공선성(Multicollinearity) | 회귀 분석에서 독립변수들 간에 강한 상관관계가 나타나는 문제 |
| VIF(분산 팽창 요인) | 5 미만 양호, 10 이상이면 다중 공선성 문제 있음 |

- 공분산은 두 변수가 같이 움직이는 정도, VIF는 그 공분산이 너무 커서 모델이 헷갈려 하는 정도를 숫자로 보여준다.

### 회귀 트리 (Decision Tree Regression)
- 결정 트리와 결정 트리 기반의 앙상블 알고리즘은 분류뿐만 아니라 회귀도 가능하다.
- 분류와 유사하게 분할을 하며, 최종 분할 후 각 분할 영역에서 실제 데이터(타겟)의 평균 값으로 학습 및 예측을 수행한다.
- 복잡한 트리 구조를 가질 경우 과적합 위험이 있으며, 트리 크기와 노드 개수 제한 등으로 개선할 수 있다.
- 데이터의 추세를 따라가는 것이 아니라 기존 영역에 가두어 버리기 때문에, 새로운 큰 값에 대해서는 범위 내 최대 평균값으로 반복 예측한다.
- 미세한 노이즈까지 학습하기 때문에 데이터 작은 변화에 매우 민감하다.
- 범주형 변수가 많거나, 독립변수 간 상관관계가 강하거나, 데이터가 곡선일 때 사용하기 좋다.

### TF-IDF

| 약어 | 의미 | 기능 |

## 7. LLM

### 7.1 ChatOpenAI 기초

#### ChatOpenAI
- OpenAI에서 개발한 대화형 AI와 상호작용하기 위한 표준화된 인터페이스를 제공한다.
- 단순 텍스트 완성이 아닌 '대화(Chat)' 형식에 최적화된 모델을 호출할 때 사용한다.

#### 주요 파라미터

| 파라미터 | 설명 |
| --- | --- |
| model_name | 사용할 AI 모델의 구체적인 명칭 |
| temperature | 0.0 ~ 2.0. 낮으면 일관적/논리적, 높으면 창의적 |

- temperature 0.1: 가장 확률 높은 답변 선택. 코딩, 사실 관계 등 정확도 중요한 작업
- temperature 0.9: 다양한 단어 선택. 소설 쓰기, 아이디어 생성 등

### LogProb (로그 확률)
- 토큰이란 문장을 구성하는 개별 단어나 문자 등의 요소를 의미하고, 확률은 모델이 그 토큰을 예측할 확률을 나타낸다.
- 특정 단어가 올 확률(0~1)에 로그를 취해 표현한다. 확률 100%는 log(1.0) = 0이며, 확률이 낮아질수록 음수로 표현된다.
- 값이 0에 가까울수록 AI가 그 단어를 선택할 확신이 높았다고 판단한다.
- 모델이 얼마나 자기 답변에 확신을 가지고 있는지 분석하는 기술적인 지표이다.

### invoke vs stream

| 구분 | invoke (일괄 처리) | stream (스트리밍) |
| --- | --- | --- |
| 작동 방식 | 답변이 100% 완성될 때까지 기다린 후 한꺼번에 반환 | 첫 단어가 생성되는 즉시 한 조각씩 실시간 전송 |
| 체감 속도 | 느림 (전체 답변 생성 시간만큼 대기) | 매우 빠름 (첫 글자 나오는 즉시 출력 시작) |
| 적합한 상황 | 결과값을 받아 후처리를 해야 하는 내부 로직 | 챗봇 UI, 긴 글 생성 등 사용자 대기 시간이 중요한 서비스 |
| 로컬 캐싱 | 쉬움 (질문-답변, 한 쌍을 매칭 가능) | 까다로움 (완료 전까지는 조각 단위라 캐싱 로직 필요) |

### 7.2 프롬프트 캐싱
프롬프트 캐싱 기능을 활용하면 반복하여 동일하게 입력으로 들어가는 토큰에 대한 비용을 아낄 수 있다.

#### 캐싱 방식 비교

| 캐싱 방식 | 저장 위치 | 매칭 방식 |
| --- | --- | --- |
| InMemoryCache | 로컬 RAM | Key 정확 일치 (dict와 유사) |
| RedisCache | Redis 서버 | Key 정확 일치 |
| RedisSemanticCache | Redis + RediSearch | 임베딩 벡터 기반 의미 일치 |

#### RedisSemanticCache 동작 흐름
1. 질문: 사용자가 질문을 입력
2. 임베딩: 허깅페이스 모델이 질문을 벡터로 변환
3. 검색: Redis 내부에 저장된 기존 질문 벡터들과 거리 비교
4. 판단: 가장 가까운 데이터와의 거리가 score_threshold 이내라면 캐시 히트
5. 반환: 캐시 히트 시 LLM 호출 없이 Redis에 저장된 답변을 즉시 반환

- RediSearch 모듈이 탑재된 redis-stack이 필요하다.
- 일반 RedisCache는 글자 일치, RedisSemanticCache는 의미 일치 방식이다.

#### Dict Stream Caching
- stream을 사용하면 합쳐지기 전에 글자가 저장되기 때문에 다 합친 다음 직접 저장해야 한다.

### 7.3 멀티모달 모델
- 기존 LLM이 텍스트라는 단일 모드만 처리했다면, 멀티모달 모델은 여러 가지 형태의 정보를 함께 처리할 수 있다.

#### CLIP (Contrastive Language-Image Pre-training)
- OpenAI에서 발표한 모델로, 텍스트와 이미지를 하나의 공간에서 연결하는 방법론이다.
- 기존 AI가 이미지를 정해진 클래스로만 배웠다면, CLIP은 자연어 설명을 통해 이미지의 맥락을 배운다.

#### 핵심 개념

| 개념 | 설명 |
| --- | --- |
| 공동 임베딩 공간(Shared Embedding Space) | 텍스트/이미지를 각각 특징 벡터로 변환 후 동일 좌표 평면 배치 |
| 텍스트 인코더 | 문장을 벡터로 변환 |
| 이미지 인코더 | 이미지를 벡터로 변환 |
| 대조 학습(Contrastive Learning) | 관련 있는 텍스트/이미지는 가까워지고, 관련 없으면 멀어지도록 학습 |

#### 벡터 유사도 (Vector Similarity)
- 두 데이터가 얼마나 비슷한지 수학적으로 계산하며, 주로 코사인 유사도(Cosine Similarity)를 사용한다.
- 유사도 1에 가까움: 정확한 설명
- 유사도 0에 가까움: 아무 상관 없음

### 7.4 LCEL (LangChain Expression Language)
- LangChain 프로토콜을 사용하여 복잡한 체인(Chain)을 더 쉽고 직관적으로 구축하기 위한 선언적 언어이다.
- 여러 구성 요소(프롬프트, 모델, 출력 파싱 등)를 리눅스의 파이프 연산자(|)와 유사한 방식으로 연결한다.

#### LCEL 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| ChatPromptTemplate | 이전 대화 내용을 기억시켜 대화가 이어지게 도와주는 템플릿 (system, human, ai) |
| OutputParser | 답변을 문자열로 리턴 (기본) |
| JsonOutputParser | JSON 형식의 답변을 dict로 리턴 |
| PydanticOutputParser | 사용자 정의 데이터 모델 규격에 맞춰 가져옴 |
| Batch | 여러 질문을 list로 받아 일괄 처리 |
| Async | 단일 쓰레드에서도 여러 요청을 효율적으로 처리 (서버 응답성 유지) |
| Parallel | 하나의 데이터를 여러 개의 template에 넣음 |
| RunnablePassthrough | 앞의 정보를 보존하면서 새로운 정보를 추가하는 전용 통로 |

#### Batch vs Parallel
- Batch: 하나의 template에 여러 개의 데이터를 넣음
- Parallel: 하나의 데이터를 여러 개의 template에 넣음

### 7.5 RAG (Retrieval-Augmented Generation)
- 생성형 AI가 학습하지 않은 최신 정보나 특정 조직의 내부 데이터를 바탕으로 답변할 수 있도록 돕는 기술이다.

#### RAG 구성 요소

| 단계 | 설명 |
| --- | --- |
| R (Retrieval) | 질문과 관련된 문서를 방대한 데이터베이스에서 검색 |
| A (Augmentation) | 검색된 정보를 질문과 결합하여 프롬프트 보강 |
| G (Generation) | 보강된 맥락을 바탕으로 최종 답변 생성 |

#### RAG 개발 순서

| 순번 | 단계 |
| --- | --- |
| 1 | 문서 불러오기 |
| 2 | 문서 분할하기 |
| 3 | 임베딩 |
| 4 | Vector DB 생성 및 저장 |
| 5 | 검색기 생성 |
| 6 | 프롬프트 생성 |
| 7 | LLM 생성 |
| 8 | 체인 실행 |

### 7.6 RAG-Anything
- 다양한 형식의 문서를 분석하여 그래프 기반의 멀티모달 지식 베이스를 구축하고, 정교한 질의응답을 수행하는 차세대 RAG 프레임워크이다.

#### 주요 기능 3단계

##### 1) Multi-modal Content Parsing

| 기능 | 설명 |
| --- | --- |
| Hierarchical Text Extraction | 문서의 제목, 본문 등 계층적 구조를 유지하며 텍스트 추출 |
| Image Caption & Metadata | 이미지 내 객체를 감지하고 캡션 생성 및 메타데이터 추출 |
| LaTeX Equation Recognition | 복잡한 수식을 기계가 읽을 수 있는 LaTeX 형식으로 변환 |
| Table Structure Parsing | 표의 행과 열 관계를 유지하며 데이터 구조화 |

##### 2) Graph-based Multi-modal Knowledge Grounding
- 텍스트 정보 처리: LLM을 통해 텍스트 내 주요 개념과 이들 간의 관계 파악
- 지식 그래프 생성: 추출된 정보를 노드와 엣지로 구성된 그래프 형태로 저장
- VLM/LLM 프로세서: 비전 모델(VLM)을 사용하여 텍스트와 이미지/표 사이의 논리적 연결 고리 생성
- Merged KG: 여러 문서에서 추출된 개별 그래프들을 하나의 거대한 지식 그래프로 통합

##### 3) Hybrid Query & Retrieval
- Graph-based Retrieval: 지식 그래프를 탐색하여 질문과 관련된 개념적 관계망 추적
- Embedding-based Retrieval: 벡터 DB를 통해 질문과 유사한 텍스트 청크 및 이미지 정보 검색
- 정보 통합: 그래프 검색 결과와 벡터 검색 결과를 결합하여 LLM에게 전달

특이사항: 표의 방향이 역방향인 경우 분석 능력이 감소할 수 있으며, 이미지보다 텍스트를 우선시하는 경향이 있다.
